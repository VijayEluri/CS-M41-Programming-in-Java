      2. Understanding null, and Java-Objects in general:
        1. An "object" in Java is a special thing. Therefore I refer to it as "Java-object".
        2. Objects of primitive data type (int's, boolean's, double's, char's, and some more) are NOT Java-objects.
        3. The speciality of Java-objects is that, unlike int's for example, you never have them "in your hands" --- only pointers.
        4. A variable for a Java-object is a pointer.
        5. That is, it is a memory address!
        6. There is a special exceptional ("singular") memory address, namely "null", which stands for something which is NOT THERE.
        7. Study the demo-program NullString carefully!

      • This is to make sure that you really concentrate on the essentials, and don't copy-and-paste code (which usually contains a lot of noise).
      • Thus under no circumstances an "import"-statement can be used.
    5. Compared to the book, we introduced the final-qualifier, which you are expected to use.
    6. Take care about precise class- and file-names.
    7. Care about private and public.
    8. Use private helper-functions (methods).
    9. Choose good variable-names.
    10. Provide a well-organised indentation.
    11. There should be no tab-symbols in your file, and no trailing spaces.
    12. There shouldn't be non-ASCII symbols in your files.
    13. No comments are allowed in the code: write good code, so that no comments are needed!
    14. Finally, follow precisely the specification.
    15. Additionally to the first coursework, now also the function-signatures (name, parameter-types and return-type) are part of the specification, so no deviation is allowed.
    16. If you use additional functions, make them private (they do not belong to the published interface).
