Coursework

  • First coursework: Deadline 9/11/2009.
  • A program "Poker.java" has to be written, consisting of the class "Poker", and this file has be sent via e-mail to me (just a single file).
  • This program shall take one command line argument N.
  • For N=0 the following should happen:
    1. A hand of (five) cards is read from standard input, where each card comes in the form "Jack of Clubs" etc. (as we had it in the example from the lecture).
    2. Additional spaces (or space symbols like end-of-line symbols or tab-symbols) must not matter.
    3. The program then determines which of the nine standard hand ranks
      1. Straight flush (a straight and a flush)
      2. Four of a kind (four equal card ranks)
      3. Full house (two equal card ranks together with three equal card ranks)
      4. Flush (all cards of same suit)
      5. Straight (cards in rank order, where the ace can be used as the lowest or the highest card rank)
      6. Three of a kind (three equal card ranks)
      7. Two pair (two times two equal card ranks)
      8. One pair (two equal card ranks)
      9. High card (none of the above).
      applies for the hand, where in case of several possibilities the higher rank has to be output.
  • For N > 0 the program chooses itself N random hands, counts how often each of the above hand ranks occurs, and outputs the relative frequency of each rank, the quotients (number of occurrences) / N.
  • You need to determine for which N the values are really close to the correct probabilities, and in a comment in your program you need to show such N and the output produced by your program.
  • Very important:
    1. Your program must compile.
    2. Your program must run.
    3. It must produce correct results.
    4. Try to give good error messages in case some user input is wrong.
    5. Use good variable names.
    6. Declare variables as local as possible.
    7. Use many functions.
    8. Specify the meaning of functions and important variables via comments.
  • Hints:
    1. Strings are not appropriate data types (except for input and output).
    2. Best is to use codes, like 1 .. 52 for the cards, 1 .. 4 for the suites, 1 .. 13 for the card ranks, and 1 .. 9 for the hand ranks.
    3. You can use modular arithmetic to handle these codes somewhat more elegantly, but this is not needed.
    4. Provide functions for extracting the suit-code and the card-rank-code from a card-code.
    5. When reading cards from (standard) input, you read in single strings, and then you need to compare the input string objects to given strings in order to compute the codes. Use the member function "String.equals" for that.
    6. Thus provide functions which convert strings for card ranks and strings for suits to the respective codes.
    7. Conversely you need functions which convert codes into the strings.
