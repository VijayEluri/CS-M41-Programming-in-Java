Coursework

First coursework

  • This coursework is paper-based, to be handed in as ordinary coursework via the coursework box.
  • Don't forget to sign and attach the coursework sheet.
  • Also state clearly the module name, your name and your student number.
  • Deadline is Monday, 15/11/2010, 16:00.
  • You are expected to have tested your code (compiled it and run it).
  • Syntax errors will result in deductions.
  • Use only the Java constructions presented in the lectures.
  • Anything else will result in a deduction of 50 percent! (The same will hold for the second coursework.)
  • This is to make sure that you really concentrate on the essentials, and don't copy-and-paste code (which usually contains a lot of noise).
  • In total you can get 100 marks for this exercise. There are three parts:
    1. Rewrite the program Shuffle from the lecture, now using your own (good!) variable names, and making constants final.
      • Another change is that no cards are shuffled, but just numbers from 1 to N, where N is to be read from the command line.
      • So the output of the program is a random permutation of the sequence 1, ..., N.
      • Try in general to make this little program as nice as possible.
      • Explain also in your own words how it works.
      • This exercise is worth 20%.
    2. Run computational experiments to check that your shuffling code works as advertised.
      • Write a program ShuffleTest that takes command-line arguments M and N, does N shuffles of an array of size M that is initialised with a[i] = i before each shuffle, and prints an M-by-M table such that row i gives the number of times i wound up in position j for all j.
      • All entries in the array should be close to N/M (if your shuffling works).
      • Explain your program.
      • This exercise is worth 50%.
    3. Bad shuffling.
      • Suppose that you choose a random integer between 0 and N-1 in our shuffling code instead of one between i and N-1.
      • Show that the resulting order is not equally likely to be one of the N! possibilities.
      • Use Run ShuffleTest for this version, with decent values of N and M (so that it takes some time, and the statistical quality is high).
      • Document your findings, and explain why this form of shuffling is bad.
      • This exercise is worth 30%.
